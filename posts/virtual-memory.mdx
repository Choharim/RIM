물리 메모리의 크기와 상관없이 프로세스에 커다란 메모리 공간을 제공하는 기술이다.

## 특징

- 가상 메모리의 최대 크기는 물리 메모리의 최대 크기로 한정되지만, 저장 장치에 있는 스왑 영역을 이용해 무한대의 크기처럼 사용할 수 있다. 즉, `가상 메모리에서 메모리 관리자가 사용할 수 있는 메모리의 전체 크기 = 물리 메모리 크기 + 스왑 영역 크기`
  - 메모리 관리자는 스왑 영역으로 물리 메모리의 부족한 부분을 보충한다.
- 가상 메모리 주소는 물리 메모리 주소와 상관없이 항상 0번지부터 시작하고 연속된 메모리 공간을 갖는다.

## 동적 주소 변환

메모리 관리자가 물리 메모리와 스왑 영역을 합쳐서 `프로세스가 사용하는 가상 주소`를 실제 메모리의 `물리 주소로 변환`하는 작업이다.

### 매핑 테이블

가상 주소와 물리 주소를 일대일로 매칭한 테이블로 관리하여, 가상 주소가 물리 메모리의 어느 위치에 있는지 알 수 있다. 메모리 관리자는 특정 프로세스가 실행될 때마다 해당 매핑 테이블을 참조해 가상 주소를 물리 주소로 변환한다.
매핑 테이블에서 페이지를 프레임과 연결시킨 페이지 테이블에는 **페이지 번호**(페이지가 순차적이지 않을 때), **플래그 비트**, **프레임 번호**(주소 필드)로 구성되어 있다. 플래그 비트에는 해당 페이지가 메모리에 올라온 후 사용된 적 있는지 알려주는 접근 비트, 해당 페이지가 메모리에 올라온 후 데이터의 변경이 있었는지 알려주는 변경 비트, 해당 페이지가 물리 메모리에 있는지 또는 `스왑 영역에 있는지 나타내는 유효 비트`, 그외 해당 페이지에 대한 `읽기/쓰기/실행 권한이 있는지 확인하는 권한 비트`로 이루어져 있다. 요구 비트가 1일 때(해당 페이지가 스왑 영역에 있을 때)의 프레임 번호에는 스왑 영역 내 페이지의 주소가 저장되어 있다.

## 메모리 분할 방식

가상 메모리 시스템에서는 메모리의 운영체제 영역을 제외한 나머지 영역을 나누어 프로세스에게 할당한다. 고정 분할 방식인 페이징과 가변 분할 방식인 세그먼테이션, 이 둘을 혼합한 세그먼테이션-페이징 혼용 기법이 존재한다.

### 고정 분할 방식 (Paging)

물리 주소 공간을 같은 크기로 나누어 사용하고 가상 주소 공간도 같은 크기로 나누어 사용한다.

<Picture src="virtual-memory.png" />

- 가상 주소의 분할된 각 영역은 페이지라 하고, 첫 번째 영역은 페이지0 두번째 영역은 페이지1 과 같이 번호를 매겨 관리한다.
- 물리 주소의 분할된 각 영역은 프레임이라 하고, 페이지와 동일하게 번호를 매겨 관리한다.
- 특정 페이지가 어느 프레임과 연결되었는지 정보가 담긴 매핑 테이블이 있다.
  물리 메모리가 아닌 스왑 영역에 있는 페이지의 경우 매핑 테이블에 invalid라 표시된다.

<aside>
❓ **가상 주소를 물리 주소로 변환하는 방법**

1. 사용자 프로세스A는 페이지1 (10 ~ 19)의 가상 메모리를 할당받았다.
2. 프로세스 A가 `가상 주소 18번지` 접근을 요구한다.
3. 메모리 관리자는 가상 주소 18번지가 프로세스 A가 할당 받은 가상 메모리 범위내 있는지 확인한다.
4. 주어진 범위에 해당한다.(범위 밖이면 인터럽트가 발생해 해당 프로세스는 강제 종료된다.)
5. 페이지 1의 8번째에 위치한다.
6. 메모리 관리자는 매핑 테이블을 이용하여 페이지1이 프레임3에 연결되어 있다는 것을 알아낸다. (+매핑 테이블을 이용해 권한을 파악하여 권한이 있을 때만 접근 가능하다.)
7. `프레임3의 8번째 위치가 물리 주소`이다.

</aside>

## 페이지 부재

프로세스가 페이지를 요청했을 때 해당 페이지 테이블에 프레임 번호가 없을 때 즉, 해당 페이지가 물리 메모리가 아닌 스왑 영역에 있을 때를 말한다.

### 페이지 교체

요구되는 페이지가 스왑 영역에 있을 때 물리 메모리로 옮겨야 사용할 수 있다. 만약 물리 메모리에 빈 프레임이 없다면 물리 메모리에 있는 프레임 중 하나를 선택해 해당 페이지를 스왑 영역으로 옮기는 스왑 아웃을 하고, 물리 메모리로 이동을 원하는 페이지를 스왑 인한다. 이때, 매핑 테이블에 요구 비트와 페이지 번호가 업데이트 된다.

이처럼 페이지가 교체되는 알고리즘에서는 지역성 이론을 고려하여 시간적으로나 공간적으로 먼 페이지가 앞으로 적게 사용될 페이지로 예상하여 스왑 아웃 될 페이지로 선정한다.

## 스레싱

메모리가 꽉차서 잦은 페이지 부재가 발생하고 페이지를 교체(스왑 인/아웃)하는 횟수가 잦아져 작업이 멈춘 것 같은 상태를 말한다.

스레싱으로 인해 속도가 느려진 컴퓨터에 물리 메모리 크기를 늘리면 스레싱 발생 지점이 늦춰져 속도가 빨라진다.

## 요구 페이징

요구 페이징은 프로세스가 요청할 때 해당 페이지를 메모리로 가져오는 방법이다.

메모리에 많은 프로세스가 올라와서 꽉차게 되면 새 프로그램을 실행하기 위해 기존 프로그램을 스왑 영역으로 옮기는 횟수가 잦아지는 등의 메모리 관리가 복잡해져 프로세스의 응답속도가 느려진다. 그리고 불필요한 프로세스까지 메모리를 차지하면 메모리가 낭비된다. 때문에 메모리에는 필요한 프로세스만 유지하는 것이 좋다.
