## 스레드

준비 Queue에 있는 프로세스를 CPU 스케줄러가 디스패치하여 CPU에 전달해줄 때, CPU가 전달받는 일 하나가 스레드이다. 그러므로 CPU가 처리하는 작업 단위는 프로세스로부터 전달받는 스레드이다.

<aside>
💡 프로세스는 운영체제의 작업 단위이고, 프로세스를 처리하기 위해 CPU가 실행하는 작업 단위가 스레드이다.

</aside>

하나의 프로세스를 한 개의 스레드로 처리할 수도, 여러개의 스레드로 나누어 처리할 수도 있다.

### 멀티 스레드

프로세스 내 작업을 여러 개의 스레드로 분할하여 작업 부담을 줄이는 프로세스 운영 기법이다. 오늘날의 운영체제는 프로세스 작업의 유연성을 얻기 위해 멀티 스레드를 사용한다.

> 장점

- 분할 작업을 통해 작업 효율이 높아진다.
  프로세스의 처리시간이 단축되고 응답 속도가 빨라진다.
- 하나의 프로세스 안에서 자원을 공유하여 자원 낭비를 줄인다.  
  ❗비슷한 작업을 위해 fork로 프로세스를 복제하여 생성하면 자원(코드, 데이터, 파일과 같은 정적 영역)이 중복된다. 프로세스 내에 스레드로 관리하면 자원을 공유하여 낭비를 막을 수 있다.

> 단점

- 프로세스 내 모든 스레드가 자원을 공유하기 때문에 한 스레드에 문제가 생기면 전체 프로세스에 영향을 준다.

### 멀티 태스킹

여러 프로세스를 동시에 실행하는 환경으로, 운영체제가 CPU에 작업을 줄 때 시간을 잘게 나누어 배분하는 기법인 시분할 시스템이 있다.

### 멀티 프로세싱

하나의 컴퓨터에 여러 개의 CPU를 두거나 하나의 CPU안에 여러 코어를 두어 여러 스레드를 동시에 처리하는 환경을 말한다.

### CPU 멀티 스레드

하나의 CPU에서 여러 스레드를 동시에 처리하는 병렬 처리 기법이다.
